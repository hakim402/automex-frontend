- `app/layout.tsx` is the Next.js root layout: it validates env at startup, registers four Google Fonts (Poppins, Cairo, Noto Sans SC, Noto Naskh Arabic) as CSS variables (`--font-en`, `--font-ar`, `--font-zh`, `--font-fa-ps`), injects `globals.css`, wraps children in `GoogleOauthProvider`, attaches `OrganizationSchema` + `LocalBusinessSchema` JSON-LD, a global `Toaster`, `WhatsAppButton`, and `GoogleAnalytics`.
- `app/[locale]/layout.tsx` is the locale-aware shell: it validates the route param against `SUPPORTED_LOCALES`, calls `setRequestLocale`, loads messages via `next-intl/server`, renders `NextIntlClientProvider`, and sets `dir="rtl"` for Arabic/Farsi/Pashto locales. `generateStaticParams` pre-renders every supported locale.
- `app/[locale]/page.tsx` composes the home page from `_components/*` sub-folders (`Header`, `HomeHero`, `AiShowcase`, `HowItWorksWrapper`, `TechStackSection`, `ConnectedModelSection`, `FooterSections`) and falls back to English content when a locale key is missing.
- `app/globals.css` centralises the design system: Tailwind v4 imports (`tailwindcss`, `tw-animate-css`, `shadcn/tailwind.css`), an `@theme inline` block mapping semantic tokens (`--color-*`) to CSS custom properties, separate `:root` / `.dark` palettes, font routing by `html[lang]`, prose styles, and brand-gradient utilities that flip direction under `html[dir="rtl"]`.
- Static-site generators live alongside the layout: `manifest.ts` (PWA manifest), `robots.ts` (disallow `/api/`, `/admin/`, `/dashboard/`), and `sitemap.ts` (duplicates every static page URL across all `SUPPORTED_LOCALES`).
- Dependency direction is one-way: layout → `_components/*` → shared UI (`@/components/ui/*`, `@/providers/*`, `@/lib/*`); components never import sibling layouts.
