Built as a Next.js App Router client page under `app/[locale]/dashboard/(routes)/security/`.
- `page.tsx` is the single client entry (`"use client"`) that owns auth gating (redirects unauthenticated users via `useAuth`), fetches a lightweight session count once for the overview card, then composes four sibling components: `SecurityOverviewCard`, `ChangePasswordForm`, `ActiveSessionsCard`, `DangerZoneCard`. A shared `SecuritySkeleton` is returned while loading or when no user is present.
- Each `_components/*.tsx` file is an independent client component with its own data fetching, local state, and optimistic UI; they share no internal state beyond props passed from the page.
- Data layer is centralized in `@/lib/auth` (`getActiveSessions`, `revokeSession`, `logout`, `getErrorMessage`, plus `Session` / `User` types); components call these functions directly rather than through a higher-level service.
- UI primitives come from the shared `@/components/ui/*` shadcn set (`Button`, `Input`, `Checkbox`, `Dialog`) and the dashboard-wide `SectionCard` wrapper from `../../profile/_components/SectionCard`; layout uses Tailwind classes and `framer-motion` + `AnimatePresence` for list transitions.
- Internationalization goes through `next-intl`'s `useTranslations("Security")`, and RTL is handled by detecting `/ar` in the pathname and applying `dir="rtl"` on the root container.
