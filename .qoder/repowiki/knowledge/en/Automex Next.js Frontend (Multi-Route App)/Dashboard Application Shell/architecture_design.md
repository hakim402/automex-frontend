The top-level `layout.tsx` is the single entry point for every dashboard page: it resolves the current locale via `next-intl/server`, toggles `dir="rtl"` for Arabic/Farsi/Pashto, wraps children in `SidebarProvider` from `@/contexts/sidebar-context`, and renders the collapsible `Sidebar` alongside `DashboardMain`. All feature pages live under `(routes)/` — a Next.js route group that shares this layout without adding URL segments — so profile, security, consulting, notifications, services, projects, and support all inherit the same chrome. The root `page.tsx` is the default landing (`GreetingBanner`) when no sub-route matches. Cross-cutting UI primitives (`Header`, `Sidebar`, `DashboardMain`, `GreetingBanner`) live in `_components/` at the dashboard root so they are reusable by every child route.
