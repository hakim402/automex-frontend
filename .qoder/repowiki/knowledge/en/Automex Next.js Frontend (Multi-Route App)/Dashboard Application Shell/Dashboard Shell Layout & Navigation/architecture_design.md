- `layout.tsx` is the Next.js App Router layout for `/[locale]/dashboard`; it reads the locale via `next-intl/server`, computes an RTL flag from a hardcoded `RTL_LOCALES` set (`ar`, `fa`, `ps`), then wraps children in `SidebarProvider` + a flex row of `<Sidebar>` (fixed drawer) and `<DashboardMain>` (content column).
- `_components/Sidebar/Sidebar.tsx` owns the left rail: two data-driven nav arrays (`MAIN_NAV`, `BOTTOM_NAV`) typed as `{ icon, labelKey, href, badge? }`, collapsed/expanded state and mobile overlay driven by `useSidebar()` from `@/contexts/sidebar-context`, with active-link detection via `usePathname().startsWith(href)`.
- `_components/Header/DashboardHeader.tsx` provides the sticky top bar (hamburger, optional page title, language switcher, theme toggle, notification bell, avatar chip) and consumes `useSidebar()`, `useAuth()`, and `useLocale()`.
- `_components/DashboardMain.tsx` is a client wrapper that mirrors the RTL decision on the pathname prefix, applies a dynamic inline `margin-left/right` equal to `sidebarWidth` on `lg:` screens, renders `<DashboardHeader />`, and exposes a scrollable `<main>` slot for page content.
- `_components/GreetingBanner.tsx` is the default page's hero card; `page.tsx` composes only this banner inside a centered container.
- Dependency direction is one-way: layout → Sidebar / DashboardMain → Header; shared UI primitives come from `@/components/ui/*`, icons from `lucide-react`, animations from `framer-motion`, routing from `@/i18n/routing`, and auth/sidebars from `@/contexts/*`. No cross-imports between sibling components except through these shared contexts.
