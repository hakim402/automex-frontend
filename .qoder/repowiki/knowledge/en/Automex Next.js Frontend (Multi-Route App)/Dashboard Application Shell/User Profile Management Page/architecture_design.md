Single-page client component (`page.tsx`) composed of four co-located React components under `_components/`: `ProfileHeader` (avatar + name/status), `PersonalInfoForm` (editable fields persisted via `updateMe()` from `@/lib/auth`), `AccountInfoCard` (read-only metadata with clipboard copy), and `ProfileSkeleton` (loading placeholder). The page owns auth gating — it reads `user`/`loading` from `useAuth()`, redirects unauthenticated users to `/sign-in`, and renders the skeleton while context hydrates. All UI text is internationalized through `next-intl`'s `useTranslations("Profile")`. Shared layout primitives `SectionCard` and `FormRow` are reused across the form and info card, keeping each section self-contained. Data flow is one-way: props descend from the page; mutations go up through `reloadUser()` in `AuthContext` plus a Sonner toast for feedback.
