Three sibling sub-packages with a clear separation of concerns:
- `components/ui/` — low-level unstyled primitives built on Radix UI (`Slot`, `AlertDialog`, `Dialog`, `Popover`, `Select`) and styled via `class-variance-authority` + `cn`; each file exports one primitive (e.g. `Button`, `Input`, `Textarea`, `Checkbox`).
- `components/shared/` — medium-complexity page-section components (`StatsRow`, `ValuesSection`, `Timeline`, `TeamSection`, `PageCTA`, `SectionHeader`, `HowItWorks`, `ConnectedModel`, `TechStack`, `SafeHTML`) re-exported through a barrel `index.ts` so consumers import from `@/components/shared`. These are pure presentational props; they never hold business data.
- `components/seo/` — thin JSON-LD injectors (`JsonLd` + typed schema wrappers: `BreadcrumbSchema`, `FAQSchema`, `LocalBusinessSchema`, `OrganizationSchema`) that render `<script type="application/ld+json">` nodes.
- `config/` — the counterpart to `shared/`: for every complex shared component there is a matching `useXxxConfig()` hook (e.g. `useConnectedModelConfig`, `useHowItWorksConfig`, `useTechStackConfig`) that calls `next-intl`'s `useTranslations` and returns a plain object conforming to the component's prop types. This keeps all copy/marketing data out of the component source.
- `WhatsAppButton.tsx` is a standalone client-only floating CTA reading `NEXT_PUBLIC_WHATSAPP_SUPPORT` at runtime.
Dependency direction is strictly `config/* → components/shared/*` (via imported prop types); `components/shared` has no dependency on `config`; `components/ui` is leaf-level with no internal deps beyond `@/lib/utils`.
