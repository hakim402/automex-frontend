Organized as a flat component library under `crm/_components/` consumed by CRM route handlers:
- `crm-shared/` is the shared sub-package: `CrmFormField.tsx` provides a uniform label/error wrapper; `schemas.ts` centralizes Zod definitions (`BUDGET_VALUES`, `TIMELINE_VALUES`, `MEETING_TYPE_VALUES`) and a `getLeadContactFields(isAuthenticated)` factory that toggles email/phone requiredness for logged-in vs guest users. Domain-specific field groups live in `crm-shared/fields/` (e.g. `BudgetTimelineFields.tsx`, `ServiceMultiSelect.tsx`) and booking inputs in `crm-shared/booking/` (`DatePicker.tsx`, `SlotPicker.tsx`).
- `hooks/useCrmFormSubmit.ts` is a generic RHF + Server Action adapter: it wraps an action returning `ActionResult<TData>`, maps server-side `fieldErrors` into `react-hook-form` errors via `UseFormSetError`, surfaces success through an optional callback, and shows `sonner` toasts on failure.
- Top-level `CrmClientPage.tsx` composes these pieces into the CRM hub layout (hero, three path cards, process steps from `ProcessStep[]`, newsletter). A thin re-export of `BudgetTimelineFields` exists at the top level for legacy consumers.
- Dependency direction is one-way: leaf components import only from `@/components/ui/*`, `next-intl`, `react-hook-form`, and sibling files inside `crm-shared`; nothing in this module imports from CRM route pages, keeping it reusable across routes.
