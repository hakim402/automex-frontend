Each sub-route (`/crm`, `/crm/quote`, `/crm/contact-sales`, `/crm/book-a-call`) follows the same layout: a thin async `page.tsx` that fetches metadata via `generateMetadata` (using `next-intl/server` + shared `generatePageMetadata`) and renders a `<XxxClientPage>` React Server Component from `_components`. All client pages import their mutations from the single `actions.ts` at the group root, which re-exports typed wrappers around `@/lib/automex/crm` calls and centralises error mapping through `handleAutomexError` into the shared `ActionResult<T>` shape. Cross-cutting UI — Zod schemas, field components (`CrmFormField`, `BudgetTimelineFields`, `ServiceMultiSelect`), the `useCrmFormSubmit` hook, and the booking `DatePicker`/`SlotPicker` — lives under `_components/crm-shared` and is consumed by every sub-route's client page, keeping each route folder minimal and focused on its own payload type.
