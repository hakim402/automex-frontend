Two Next.js route segments under app/[locale]/(routes)/services/:
- /page.tsx (listing): server component that reads params.locale and searchParams.category, calls fetchServices/fetchServiceCategories/fetchTechnologies/fetchIndustries from @/lib/automex/content in parallel, builds a BreadcrumbSchema + ItemList JSON-LD, then delegates rendering to the client component _components/ServicesClientPage.tsx. Category filtering is URL-driven via Link query params; infinite scroll is implemented through a Server Action.
- /[slug]/page.tsx (detail): server component that fetches the service by slug, returns empty Metadata when not found, otherwise constructs OpenGraph/Twitter metadata from the service's seo fields, builds a Service JSON-LD, computes up to three related services from the same category, and renders _components/ServiceDetailClient.tsx.
- actions.ts declares a single use server function loadMoreServicesAction(category, page, locale) returning the shared ActionResult<T> envelope used by the listing client to append pages without navigation.
- Both client components are marked 'use client' and consume next-intl (useTranslations('ServicesPage' | 'ServiceDetail')) plus the shared @/i18n/routing.Link for typed routing. Data fetching is isolated to server components and the action; clients receive fully hydrated props.
