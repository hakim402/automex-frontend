All children are sibling route groups inside `app/[locale]/(routes)`, sharing the same Next.js App Router layout and locale resolution. The `crm/` subtree is a nested route group whose top-level `page.tsx` acts as a shell delegating to sub-routes (`book-a-call`, `contact-sales`, `quote`) via `<Link>`; each sub-route owns its own `_components/<XxxClientPage>.tsx` client component plus an optional `actions.ts`. The `services/` subtree mirrors this shape with a list page and a `[slug]/detail` dynamic segment. Static pages (`about`, `contact`, `privacy`) follow the same per-page `page.tsx` + `_components/ClientPage` split. Cross-child shared UI lives in `crm/_components/crm-shared/` (form fields, schemas, hooks) and is imported by both CRM sub-pages and the services actions where needed.
