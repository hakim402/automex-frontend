Two cooperating layers under `app/[locale]/(routes)/crm/_components/`:
- Client layer (`"use client"`) — reusable form primitives in `crm-shared/`: a generic `CrmFormField` wrapper, typed select fields (`BudgetTimelineFields`, `ServiceMultiSelect`), an authenticated-email field, a native `<input type="date">` `DatePicker`, a grid-based `SlotPicker`, plus the `getLeadContactFields` Zod factory in `schemas.ts`. A `hooks/useCrmFormSubmit.ts` hook wraps React transitions + Sonner toast + react-hook-form `setError` against any action returning `ActionResult<T>`.
- Server layer (`actions.ts`, marked `"use server"`) — a single file exporting one action per CRM route (`subscribeNewsletterAction`, `submitContactLeadAction`, `submitQuoteRequestAction`, `fetchAvailabilityAction`, `submitConsultationBookingAction`). Each action delegates to `@/lib/automex/crm`, maps `AutomexApiError` variants into a uniform `{ success, message, fieldErrors }` via `handleAutomexError`, and is typed with `Omit<Input, "language">` plus `SupportedLocale` so the client never sends the language field.
- The hub page component `CrmClientPage.tsx` composes these pieces (it renders `NewsletterForm` from `crm-shared` and drives the three CRM sub-routes). Dependency direction is strictly client → server-action → `@/lib/automex/*`; no cross-imports between sibling components.
