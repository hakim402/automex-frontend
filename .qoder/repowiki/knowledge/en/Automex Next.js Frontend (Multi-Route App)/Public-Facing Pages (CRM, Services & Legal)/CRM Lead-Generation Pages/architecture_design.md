Each route under `app/[locale]/(routes)/crm/` follows the same split: an async Server Component `page.tsx` handles i18n metadata via `generatePageMetadata` + `getTranslations`, fetches content from `@/lib/automex/content` (`fetchServices`, `fetchIndustries`, `fetchProcessSteps`), then delegates rendering to a co-located `_components/*ClientPage.tsx` client component. The three form routes (`contact-sales`, `book-a-call`, `quote`) accept an optional `service` search param and pass it as a default selection. All form submissions are routed through the single server-actions file at `actions.ts`, which is also where the shared form primitives live in `_components/crm-shared/` — so the page layer stays thin while validation, schemas, and backend dispatch are centralized.
