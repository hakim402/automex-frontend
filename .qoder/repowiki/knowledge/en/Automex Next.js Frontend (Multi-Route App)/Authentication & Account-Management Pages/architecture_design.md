Organized as a Next.js Route Group `(auth)` under `app/[locale]/`, so every route is locale-aware. A single server-side `layout.tsx` resolves the current locale via `next-intl/server`, switches `dir="rtl"` for Arabic/Pashto/Farsi, and wraps children in a centered card layout with an ambient radial gradient background.

Each auth flow is a client page (`"use client"`) exporting a default React component: `sign-in`, `sign-up`, `forgot-password`, plus three sub-flows under `auth/` (`magic-link`, `reset-password`, `verify-email`). Shared visual building blocks live in `_components/` (`AuthCard`, `AuthHeader`, `AuthFormField`, `PasswordInput`, `GoogleButton`, `AuthDivider`, `AuthFooterLink`, `BackToHome`) and are reused across every page to keep the card + form chrome consistent.

Data layering is shallow: pages call functions from `@/lib/auth` (`login`, `register`, `requestPasswordReset`, `verifyMagicLink`, `getMe`, `getErrorMessage`) and update global state through `@/contexts/AuthContext`. Validation schemas are defined inline per page using Zod and wired into `react-hook-form` via `@hookform/resolvers/zod`. All user-facing strings go through `useTranslations("Auth")` from `next-intl`, and navigation uses the i18n-aware `Link` from `@/i18n/routing` (with `router.push` only for programmatic redirects). Success/error states are rendered locally inside each page rather than via shared error pages.
