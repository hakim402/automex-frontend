The app is a single Next.js project whose children are organized as App Router route groups under `app/[locale]/(routes)`, `app/[locale]/auth`, and `app/[locale]/dashboard`. Cross-child orchestration happens at three layers:
- Middleware (`proxy.ts`) composes next-intl locale detection with cookie-based auth guards, maintaining the canonical lists of PROTECTED_ROUTES, AUTH_ROUTES, and PUBLIC_ROUTES so every child route group shares one source of truth for access control.
- Root layout shell (`root_layout_assets`) wraps all routes with providers from `providers_and_contexts` (theme, Google OAuth, auth state), while `i18n/routing` plus message bundles feed both the layout and the middleware.
- Shared runtime lives in `lib_and_api` (client API client, Zod schemas, environment/locale helpers) consumed by every feature module; server Route Handlers there back the contact-form email delivery and session-cookie sync endpoints referenced by the middleware.
- Build-time wiring: `next.config.ts` enables next-intl and whitelists Unsplash + `cdn.automex.tech` for `<Image>` optimization; `package.json` exposes `generate-types` to keep generated OpenAPI types in sync with the backend schema used by all route handlers.
