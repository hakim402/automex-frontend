### AUTOMEX Django + DRF backend
- Two distinct auth schemes are used:
  - Public content endpoints under `/api/v1/` are gated by an `X-API-Key` header set from `AUTOMEX_API_KEY` (`lib/automex/env.server.ts`, `client.ts`).
  - Authenticated user endpoints under `/auth/*` use JWT access+refresh tokens stored in browser `localStorage` and refreshed proactively/reactively (`lib/api.ts` `authRequest`, `lib/auth.ts`).
- All public read endpoints return either a plain array or a DRF-style paginated envelope `{ count, next, previous, results }`; the frontend normalises both via `unwrapPaginated`.
- Error responses are normalised to `{ detail: string } | { non_field_errors: [...] } | { field: [...] }` shapes consumed by `buildApiError`.
- Verify exact endpoint paths and response schemas against the OpenAPI schema generated by `openapi-typescript https://api.automex.tech/api/schema`.
